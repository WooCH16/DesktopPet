## 데이터 흐름

```
[시작]
  └─► WindowManager.Awake()
        ├─ SetWindowLong()       → 투명·클릭통과·Topmost 적용
        └─ SetLayeredWindowAttributes(Magenta)

[매 프레임]
  └─► PetController.Update()
        ├─ [드래그 중?] ──YES──► DragController.HandleDrag()
        │                          └─ 마우스 위치 → SetWindowPos()
        └─ [자율 이동] ──────────► MovementController.Update()
                                    ├─ 경계 체크 → 방향 반전
                                    ├─ 상태 결정 (Idle/Walk/Run)
                                    └─ AnimationStateMachine.SetState()

[클릭 이벤트]
  └─► InteractionController.OnMouseDown()
        └─ AnimationStateMachine.SetState(반응 애니메이션)

[트레이 이벤트]
  └─► TrayIconManager
        ├─ 우클릭 → ContextMenu 표시
        └─ 종료 선택 → Application.Quit()
```

## CAST 힌트

- **Q1. UI 인터랙션이 3개 이상인가?**
  → YES: 드래그, 클릭 반응, 트레이 메뉴, 경계 감지 = 4개
  → UX 전문가 후보

- **Q2. 외부 API/서드파티 연동이 있는가?**
  → YES: WinAPI P/Invoke (SetWindowLong, SetLayeredWindowAttributes, SetWindowPos)
  → 통합 전문가 후보 (단, WinAPI는 단순 P/Invoke라 통합 복잡도 낮음)

- **Q3. DB 관계가 3개 이상인가?**
  → NO: 영구 저장 없음 (설정은 레지스트리 1개)

> 투입 추천: **UX 전문가** (인터랙션 복잡도 높음)
> WinAPI는 기존 v2 구현 경험 있어 통합 전문가 불필요
